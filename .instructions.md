# Skill: anti-ai-slop-designs

## Description
Apply polished, human-quality UI design principles to avoid common "AI slop" patterns — low-effort, generic, inaccessible, or visually inconsistent UI generated without human polish. Enforces intentional design, accessibility, design token consistency, and visual refinement across all frontend work.

## Triggers (when this skill activates)
This skill applies when working on:
- UI components in `src/app/components/` or `src/app/components/ui/`
- Pages in `src/app/pages/`
- Design suggestions, mockups, or layout proposals
- SCSS/CSS/Tailwind utility class styling
- Form, modal, card, list, or dashboard designs
- Any code generation that produces visible UI output

## Rules (enforced)

### 1. Design Consistency
- Use existing design tokens, spacing scale, and typography.
- Do NOT introduce ad-hoc sizes, colors, or fonts.
- Reference CSS custom properties (e.g., `--color-primary`, `--radius-md`, `--spacing-4`) instead of hardcoded values.
- Maintain a consistent border-radius, shadow, and color palette across components.

### 2. Accessibility (WCAG AA)
- Ensure **contrast ratio ≥ 4.5:1** for normal text, ≥ 3:1 for large text.
- Interactive elements must have **visible keyboard focus** indicators.
- All images must have **descriptive alt text** (not empty or generic).
- Forms must have **associated labels** (not placeholders as labels).
- Use semantic HTML (`<button>`, `<nav>`, `<header>`, etc.) over generic `<div>`s.

### 3. Intentional Layout
- Avoid filler placeholders, repetitive generic cards, and unstructured stacking.
- Prefer **meaningful hierarchy** — clear visual distinction between headings, body, and actions.
- Use consistent alignment and spacing. No "floating" elements with arbitrary margins.
- Limit card decks to **meaningful content** — avoid 12 identical-looking cards with lorem ipsum.

### 4. Visual Refinement
- Replace bland gradients, poorly aligned icons, and inconsistent radii with polished variants from the design system.
- Icons should be **consistently sized** (e.g., all 16px, 20px, or 24px — not mixed).
- Avoid visual "noise": unnecessary borders, excessive shadows, competing colors.
- Empty/loading/error states must be **designed**, not afterthoughts.

### 5. Avoid Excessive Gradients
- Minimize heavy gradient usage, especially purple→pink transitions.
- Prefer **tokenized solid colors** or subtle, purposeful gradients that align with the design system.
- If a gradient is used, it must serve a clear purpose (e.g., brand hero section, not decorative buttons).

### 6. Performance & Asset Size
- Prefer **SVG** or optimized images over heavy PNGs/JPEGs.
- Do NOT add uncompressed large assets or heavy animations by default.
- Animations should be **purposeful** (loading indicators, micro-interactions) and use `prefers-reduced-motion` respect.
- Lazy-load images below the fold.

### 7. Content Quality
- Replace generic lorem/placeholder copy with **realistic, context-aware copy** or clear `// TODO:` comments explaining required copy.
- Button labels must be **action-oriented** ("Save Changes", "Create Campaign") not generic ("Click Here", "Submit").
- Error messages must be **human-readable** and helpful, not technical stack traces.

## Examples (before → after)

| Slop (Avoid) | Polished (Goal) |
|---|---|
| Generic card with placeholder text | Concise card with clear title, 2-line body, CTA, and accessible label |
| Random color + 32px radius | Tokenized color (`--color-card-bg`) + token radius (`--radius-md`) |
| Purple→pink gradient button | Solid `--color-primary` button with subtle hover state |
| 12 identical lorem ipsum cards | 3-4 meaningful cards with varied, realistic content |
| `<div onClick={...}>` | `<button onClick={...}>` with focus styles |
| Placeholder="Enter name" as only label | Proper `<label>` + `<input>` pairing |

## Compliance Checklist
Include this checklist with each UI change to verify anti-slop compliance:

- [ ] Design tokens used (no ad-hoc colors/sizes)
- [ ] Contrast OK (WCAG AA ≥ 4.5:1)
- [ ] Alt text present (descriptive, not empty)
- [ ] Keyboard focus indicators visible
- [ ] Semantic HTML used (buttons, labels, nav)
- [ ] No excessive gradients (solid tokens preferred)
- [ ] No placeholder/filler content (realistic copy or TODO)
- [ ] Performance note (SVG/optimized assets, lazy loading)
- [ ] Consistent spacing & alignment
