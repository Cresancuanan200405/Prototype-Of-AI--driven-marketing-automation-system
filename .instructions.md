**Agent Instructions — Apply Anti-AI-Slop Designs**

Purpose
- Ensure UI work and suggested designs avoid common "AI slop": low-effort, generic, inaccessible, or visually inconsistent UI patterns generated without human polish.

Scope
- Applies to UI components, pages, and design suggestions in `src/app/components` and `src/app/pages`.
- Technologies: React + TypeScript, CSS/Tailwind/utility classes, static assets and design tokens.

Rules (enforced)
- **Design Consistency**: Use existing design tokens, spacing scale, and typography. Do not introduce ad-hoc sizes, colors, or fonts.
- **Accessibility**: Ensure contrast meets WCAG AA, interactive elements have keyboard focus, and images include descriptive alt text.
- **Intentional Layout**: Avoid filler placeholders, repetitive generic cards, and unstructured stacking. Prefer meaningful hierarchy, alignment, and spacing.
- **Visual Refinement**: Replace bland gradients, poorly aligned icons, and inconsistent radii with polished variants from the design system.
- **Avoid Excessive Gradients**: Minimize heavy gradient usage, especially purple→pink transitions; prefer tokenized solid colors or subtle, purposeful gradients that align with the design system.
- **Performance & Size**: Prefer SVG or optimized images; don't add uncompressed large assets or heavy animations by default.
- **Content Quality**: Replace generic lorem/placeholder copy with realistic, context-aware copy or clear TODOs explaining required copy.

Examples (before → after)
- Generic card with placeholder text → concise card with clear title, 2-line body, CTA, and accessible label.
- Random color + 32px radius → use tokenized color (`--color-card-bg`) + token radius (`--radius-md`).

How the agent should apply these
- When proposing UI code or mocks, validate against the rules above and list any assumptions.
- Provide a short checklist with each change: `Design tokens used`, `Contrast OK`, `Alt text present`, `Performance note`.
- If a suggested design violates a rule, propose a corrected alternative rather than leaving it as-is.

Prompts to test this instruction
- "Create a polished profile card component for `Profile.tsx` following anti-AI-slop rules." 
- "Refactor the landing hero to use design tokens, ensure WCAG AA contrast, and remove placeholder copy."
